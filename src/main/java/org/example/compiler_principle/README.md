# 编译原理专题

计算机与软件领域最核心的基础理论课之一。研究「如何将高级语言翻译成机器可执行的指令」，贯穿从源码到运行的完整链路。

> 本模块暂无代码，后续视学习进度补充。当前以知识地图形式记录核心体系，方便对照 Java 实践理解。

---

## 编译器完整流水线

```
源代码（.java / .c / ...）
    ↓
【前端】词法分析（Lexer）      → Token 流（关键字/标识符/运算符/字面量）
    ↓
【前端】语法分析（Parser）     → 抽象语法树（AST）
    ↓
【前端】语义分析              → 类型检查 / 符号表 / 作用域分析
    ↓
【中端】中间代码生成（IR）     → 三地址码 / SSA 形式
    ↓
【中端】代码优化              → 常量折叠 / 死代码消除 / 循环优化 / 内联
    ↓
【后端】目标代码生成          → 汇编 / 字节码（.class）/ 机器码
    ↓
可执行程序 / JVM 字节码
```

## 核心知识点

| 阶段 | 核心概念 |
|------|---------|
| **词法分析** | 正则表达式、有限自动机（DFA/NFA）、Token 分类 |
| **语法分析** | 上下文无关文法（CFG）、LL(1)/LR(1) 分析、递归下降、AST |
| **语义分析** | 类型系统、符号表、作用域、类型推断 |
| **中间表示** | 三地址码、静态单赋值（SSA）、控制流图（CFG） |
| **代码优化** | 常量传播、内联展开、循环展开、逃逸分析、尾调用优化 |
| **代码生成** | 寄存器分配、指令选择、指令调度 |
| **运行时** | 内存布局（栈帧/堆）、垃圾回收、调用约定 |

## 与 Java 的关联

- `javac` 前端编译 → 词法→语法→语义分析，将 `.java` 编译为 `.class` 字节码
- JIT（C1/C2 编译器）→ 运行期将热点字节码编译为机器码，含大量中端优化（逃逸分析/内联）
- 注解处理器（APT）→ 在语义分析阶段介入，Lombok/MapStruct 正是此原理
- `invokedynamic` 指令 → 编译期推迟方法绑定，Lambda / 动态语言支持的底层
- `String` 常量池 / `intern()` → 编译器字面量合并，与符号表机制一脉相承
- 逃逸分析 → JVM 判断对象是否需要堆分配，影响 GC 压力

## 与 LLM / AI 编程工具的关联

> 理解编译原理是理解 AI 如何「读懂」代码的重要基础：
>
> - LLM 的 Tokenizer → 与词法分析的 Lexer 高度类似，都在做「切词」
> - 代码补全 / AST 感知 → 很多 IDE 插件直接解析 AST，理解上下文结构
> - 静态分析工具（SpotBugs / Checkstyle）→ 遍历 AST 做规则检查
> - 编译器优化思想 → 理解 AI Coding Harness 如何「优化」生成代码的逻辑

## 推荐学习路径

```
先掌握：数据结构（树/图/哈希表） + 算法（DFA/递归）
    ↓
经典教材：龙书（Compilers: Principles, Techniques & Tools）前5章
         虎书（Modern Compiler Implementation in Java）更贴近 Java 实践
    ↓
动手实践：用 Java 实现一个简单的四则运算解释器（词法→语法→求值）
    ↓
深入 JVM：《深入理解 Java 虚拟机》第10~11章（前端编译 + JIT）
    ↓
拓展：LLVM 架构了解 / GraalVM 原生编译（AOT）
```

建议在掌握 JVM / 多线程 / IO 体系后再系统学习，收益最大。
